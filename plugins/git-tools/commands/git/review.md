name: git:review
description: Review changes before commit
---

現在のリポジトリの変更を、以下の観点でレビューしてください。

1. **デバッグコードの残存**
   - 言語に応じたデバッグ出力（`console.log`, `print`, `println!`, `dbg!` など）
   - デバッグ用の一時コードや条件分岐

2. **コメントの精査**
   - TODO / FIXME / HACK などの未解決タグ
   - コメントアウトされたコード
   - 変更内容と無関係な古いコメント

3. **コード品質**
   - 不要な空白行・trailing whitespace・フォーマット崩れ
   - 未使用の import・変数・関数

4. **機密情報・セキュリティ**
   - API キー、トークン、パスワード、DB 接続文字列等のハードコード
   - `.env` / 鍵ファイル / 認証設定ファイルの混入
   - 検出対象と対応手順の詳細は `${CLAUDE_PLUGIN_ROOT}/skills/git-helper/reference.md` §3 を参照

5. **変更内容の整合性**
   - 変更の目的と実際の差分が一致しているか
   - 意図しないファイル・行が含まれていないか

レビュー結果をまとめて提示し、問題があれば該当箇所を示して修正提案を行い、問題がなければコミット可である旨を伝えてください。
